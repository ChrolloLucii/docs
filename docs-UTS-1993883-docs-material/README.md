# Документация проекта (SMGOZ docs & pages)

## Назначение проекта

Проект предназначен для автоматической сборки и публикации статического сайта **GitLab Pages** на основе содержимого **Wiki** текущего проекта. Сборка осуществляется с помощью генератора документации `mkdocs` с использованием темы `mkdocs-material`.

* **Исходная Wiki:** [Ссылка на Wiki](https://ahcode.headoffice.psbank.local/psb-private/devops-infrastructure/opensource-admins/projects/smgoz/docs/-/wikis/home)
* **Опубликованный сайт Pages:** [Ссылка на Pages](https://psb-private.pages.headoffice.psbank.local/devops-infrastructure/opensource-admins/projects/smgoz/docs/#)

---

## Архитектура и визуальное оформление

* **Базовый образ:** Сборка и генерация сайта происходят в контейнере **Ред ОС 7.3.4**.
* **Тема оформления:** Используется кастомизированная тема **Material for MkDocs**
* **Режимы отображения:** Встроена поддержка переключения цветовых схем
* **Стилизация:** Для точечной настройки интерфейса используется кастомный CSS-шаблон `stylesheets/extra.css`.

---

## Условия и автоматизация запуска пайплайна

1. **Автоматический запуск (Scheduler):** Для исключения ручной сборки и поддержания актуальности документации настроен ежедневный запуск пайплайна по расписанию
2. **Ручной запуск:** Для сборки сайта используется специализированная джоба `pages`. Дополнительные переменные окружения при запуске указывать не требуется.

---

## Этапы подготовки и сборки сайта (`.gitlab-ci.yml`)

Процесс сборки разделен на два ключевых этапа: подготовка контента и компиляция HTML.

### 1. Подготовка данных
* Выполняется клонирование исходного проекта Wiki.
* Создается рабочая директория `docs`, куда копируется структура файлов Wiki и кастомные стили CSS.
* Стартовая страница Wiki (`home.md`) переименовывается в `index.md`, так как `mkdocs` воспринимает её в качестве главной страницы сайта.
* Запускается Python-скрипт `script/convert.py`, который парсит `.md` файлы и конвертирует специфичные теги в валидный HTML-код для корректного рендеринга.

### 2. Компиляция и публикация
* Инструмент `mkdocs` обрабатывает директорию `docs` и компилирует статическую версию сайта.
* Результат сборки складывается в директорию `$CI_PROJECT_DIR/public`.
* Содержимое папки `public` передается в GitLab Pages в качестве финального артефакта сборки.

## Поддержка

* **Проблемы с отображением:** Если на итоговом сайте обнаруживаются неотображаемые элементы, битые ссылки или некорректная разметка, необходимо доработать регулярные выражения или логику конвертации в скрипте `script/convert.py`.
